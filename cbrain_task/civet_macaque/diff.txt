

J'ai aussi ajoute le MP2RAGE, une sequence speciale du scanner pour laquelle
on n'a pas besoin de N3 -- c'est fait sur le scanner. Meme si je n'ai
pas transfere sur graham, tu peux commencer a configurer l'interface.

Nom de l'outil: CIVET-macaque

Options:
   -nifti                Output files in nifti format.
   -nonifti              opposite of -nifti [default]

   -template <val>       Define the template for image processing in
                         stereotaxic space (human: 0.50, 0.75, 1.00, 1.50,
                         2.00, 3.00, 4.00, 6.00 [default 1.00];

                         macaque: 0.25,0.50 [default 0.25]; chimpanzee: 0.50).

   -model <model>        Define the model for image-processing:
   					          	 "colin27" (MNI Colin27 asymmetric (2009)),
   					          	 "icbm152nl_09s" (MNI ICBM152 non-linear symmetric (2009a)),
                         "icbm152nl" (MNI ICBM152 non-linear 6th generation),
                         "icbm152l" in" (MNI ICBM152 linear),
                         "ibis-v24" (MNI IBIS 24 months, symmetric (2015)),
                         "ADNIhires" (MNI ADNI non-linear hi-res sym 0.5mm),

                         "NMTmacaque" (NMT macaque non-linear hi-res asym 0.25mm)
                         "NCBRchimp" (NCBR chimpanzee)  [default: icbm152nl_09s]

   -surfreg-model <model>
                         Define the model for surface registration:
                         "icbm152MCsym" (ICBM152, marching-cubes, symmetric (2014)),
                         "colinMCasym" (Colin, marching-cubes, asymmetric (2014)),
                         "samirMCasym" (IBIS Phantom (Samir), marching-cubes, asymmetric (2014)),

                         "NMTmacaque" (NMT macaque, marching-cubes, symmetric (2020))
                         "NCBRchimp" (NCBR chimpanzee, marching-cubes,symmetric (2020))

   -surface-atlas <model>
                         Define the atlas for surface parcellation:
                         "lobes" (coarse lobar parcellation, symmetric),
                         "AAL" (AAL parcellation, asymmetric, based on Colin brain),
                         "DKT" (DKT-40 parcellation, asymmetric),

                         "D99"    (D99 parcellation for macaque, symmetric),
                         "CHARM1" (CHARM Level-1 (N3D4)   parcellation for macaque, symmetric),
                         "CHARM2" (CHARM Level-2 (N3D17)  parcellation for macaque, symmetric),
                         "CHARM3" (CHARM Level-3 (N3D36)  parcellation for macaque, symmetric),
                         "CHARM4" (CHARM Level-4 (N3D52)  parcellation for macaque, symmetric),
                         "CHARM5" (CHARM Level-5 (N3D84)  parcellation for macaque, symmetric),
                         "CHARM6" (CHARM Level-6 (N3D127) parcellation for macaque, symmetric)  [default:lobes]

   -headheight <dist>    head height in mm for neck cropping (175mm for human,

						 100mm for chimp, 50mm for macaque, or use 0 for none).

------------------------------------------------------------------------------------
                         NEW:

   -pre-masked           T1 image already skull masked, with background3D0.
                         Mask will be deduced from thresholding if no
                         user-mask given.
   -no-pre-masked        opposite of -pre-masked [default]


   NEW???:
   -gaussian-curvature   produce Gaussian curvature maps on surfaces (blurred like thickness)
   -no-gaussianmean-curv opposite of -gaussian-curvature [default]


   Pour l'humain celui:

   on desactive ANIMAL dans CIVET-macaque.

   -animal               run volumetric ANIMAL segmentation (human model only)
                         [default -no-animal]
   -no-animal            don't run volumetric ANIMAL segmentation

   -lobe_atlas <model>   Use lobe atlas for ANIMAL segmentation (mandatory with -animal):
                         "icbm152nl-VI" (ICBM152 generation VI symmetric model),
                         "icbm152nl-2009a" (ICBM152 2009a symmetric model)

------------------------------------------------------------------------------------

Pour le macaque specifiquement:

-template 0.25 ou 0.50 (defaut 0.25)

- model NMTmacaque

-surfreg-model NMTmacaque

-surface-atlas {lobes, D99, CHARM1, CHARM2, CHARM3, CHARM4, CHARM5, CHARM6}
 (default=3Dlobes, tu as la description dans le help ci-haut)

-headheight 50 (si tu mets 0, ca ne fera pas de cropping; si tu mets rien,
 ca prendra le defaut qui est 50; ou tu entres la valeur de ton choix. Le
 mieux c'est de ne rien mettre et le defaut va toujours marcher.)

-pre-masked (c'est si le t1 est deja masque -- nouveau)


CIVET-macaque est capable de detecter les inputs en NIfTI ou MINC. Les
types de fichiers sont: .mnc, .mnc.gz, .mnc.Z, .nii, .nii.gz, .nii.Z.

L'option -nifti est pour produire les outputs en .nii et ca n'a rien a
voir avec les inputs en NIfTI. Pas besoin de specifier -nifti si les
inputs sont en NIfTI. CIVET va le deduire. C'est juste pour les outputs.
Aussi, un input en NIfTI n'impliquera pas des outputs en NIfTI, sauf
si on demande -nifti.

Pour le MP2RAGE, il pourrait y en avoir. Tu peux le configurer maintenant
et ca va etre bon pour CIVET-humain plus tard. Les fichiers d'inputs dans
CIVET ont la forme: prefix_t1.mnc, prefix_t2.mnc, prefix_pd.mnc,
prefix_mask.mnc. Ajoute simplement prefix_mp2.mnc. Pour utiliser mp2,
il faut toujours un t1 qui vient avec. On peut avoir un t1 sans mp2,
mais pas de mp2 sans t1. On place le mp2 au meme endroit que les autres
inputs.
